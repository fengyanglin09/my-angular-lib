MockStore Component Tests

MockStore lets a component test control selector values without creating the full real NgRx store. This is useful when the test is about rendering and dispatched actions, not reducer behavior.

Why MockStore Exists

How The Test Reads

  1. Mock selector values describe the store state the component should see.
  2. The component renders those values through normal template bindings.
  3. The test clicks a button just like a user would.
  4. The dispatch spy checks that the component sent the right action.
  5. The reducer is not tested here because reducer tests already cover state changes.
