Observable Basics

An Observable is a recipe for producing values over time. The recipe does not run when you create it. It runs when something subscribes.

What To Notice

  1. Creating an Observable does not start the work.
  2. subscribe() starts the producer.
  3. An Observable can emit one value, many values, or no values.
  4. next handles emitted values.
  5. complete means the Observable finished normally.
  6. unsubscribe() stops listening and runs cleanup.
