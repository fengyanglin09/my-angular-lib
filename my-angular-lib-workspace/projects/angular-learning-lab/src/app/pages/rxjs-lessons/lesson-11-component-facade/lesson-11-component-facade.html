Component Facade

A facade is a small service that gives the component a clean API. The component calls methods, and the facade owns Subjects, backend requests, filtering, selection, and the final vm$.

What To Remember

  1. A facade can combine server state and UI state into one vm$.
  2. BehaviorSubject is useful for local state that always has a current value, like search text or selected id.
  3. The component becomes easier to read because it calls methods instead of wiring operators directly.
  4. This pattern is similar in spirit to NgRx facades, but it can stay local to one screen.
