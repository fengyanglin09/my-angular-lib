View Model Selectors

A view model selector combines many small selectors into one object for a screen. The component selects one signal, and the template renders from that single view model.

Why This Helps

  1. The component selects one screen-shaped object.
  2. The selector composes smaller selectors, so the logic stays testable.
  3. The template reads clear names like vm().canGoNext and vm().selectedBook.
  4. The component avoids juggling many independent selector signals.
