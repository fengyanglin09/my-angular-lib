Effect Concurrency

Effects listen to actions and usually call APIs. When the same action happens several times quickly, the RxJS operator decides whether requests cancel, queue, run together, or get ignored.

How To Choose

  • Use switchMap when only the latest result should update the store.
  • Use concatMap when every request must finish in the order it was sent.
  • Use mergeMap when requests are independent and can run in parallel.
  • Use exhaustMap when repeated clicks should be ignored until the first request finishes.
