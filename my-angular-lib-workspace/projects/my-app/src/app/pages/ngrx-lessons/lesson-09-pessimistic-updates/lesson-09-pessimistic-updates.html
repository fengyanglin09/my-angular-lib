NgRx Lesson 9

Pessimistic Updates

This lesson waits for the backend before changing Entity state. The UI is slower than optimistic updates, but the reducer is simpler because it only stores data the backend has accepted.

New Idea In This Lesson

  1. The create or update action sets a pending flag, but does not change the item yet.
  2. The effect sends the request to the backend.
  3. The backend returns the accepted draft with its real id and values.
  4. The success action writes the backend result into Entity state.
  5. The failure action stores an error because no rollback is needed.
