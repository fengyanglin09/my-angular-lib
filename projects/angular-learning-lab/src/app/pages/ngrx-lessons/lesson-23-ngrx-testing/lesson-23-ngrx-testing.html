Testing Reducers, Selectors, And Effects

NgRx tests are strongest when each test focuses on one layer. Reducers are plain functions, selectors test derived data, and effects test action in to action out.

How To Think About NgRx Tests

  1. Reducer tests ask: given state and action, what next state comes out?
  2. Selector tests ask: given state inputs, what derived value comes out?
  3. Effect tests ask: given an action and mocked service response, what action is emitted?
  4. Component tests are usually separate; they verify rendering and user interaction.
