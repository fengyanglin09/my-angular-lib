NgRx Lesson 8

Optimistic Updates

This lesson updates the UI before the backend responds. If the request succeeds, the optimistic state is confirmed. If it fails, the reducer rolls the state back.

New Idea In This Lesson

  1. The first action updates state immediately in the reducer.
  2. The effect sends the request to the backend.
  3. A success action clears the pending flag or confirms the saved item.
  4. A failure action rolls the optimistic change back.
  5. This gives fast UI, but requires careful rollback data.
