Auth User State

Auth user state is a strong use case for global NgRx Store. Many screens may need to know whether the user is logged in, who they are, and what role or permissions they have.

Why Auth Belongs In Global State

  1. Many features need the same logged-in user.
  2. Effects are a clean place for login, logout, and session API calls.
  3. Selectors give components simple answers like isLoggedIn.
  4. Reducers keep success, failure, loading, and logout transitions predictable.
  5. Check session asks the backend whether a saved session still exists.
  6. Later, route guards and permissions can read the same auth state.
