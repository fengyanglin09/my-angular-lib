Query Params

Query params are optional URL values after ?. They are useful for page state like filters, sorting, search terms, selected tabs, and pagination because the same route can stay loaded while the query string changes.

What To Remember

  1. Route params identify the main thing: /projects/project-101.
  2. Query params describe how to view it: ?filter=active&sort=name.
  3. [queryParams] writes query params when navigating.
  4. queryParamMap is the stream to watch when query params can change on the same component.
