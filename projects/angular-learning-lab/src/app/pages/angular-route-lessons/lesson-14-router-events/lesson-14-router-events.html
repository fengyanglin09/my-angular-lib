Typical Successful Sequence

  1. NavigationStart
  2. RoutesRecognized
  3. GuardsCheckStart
  4. GuardsCheckEnd
  5. ResolveStart and ResolveEnd, if resolvers exist
  6. ActivationStart and ActivationEnd
  7. NavigationEnd

Mental Model

navigation is a timeline
  start
  match routes
  check guards
  run resolvers
  activate components
  finish or cancel

Router.events lets you watch that timeline.

What To Remember

  1. Router.events is an Observable of navigation lifecycle events.
  2. Use it for loading indicators, analytics, debugging redirects, and learning route sequence.
  3. GuardsCheckStart and GuardsCheckEnd surround guard checks.
  4. ResolveStart and ResolveEnd appear when a route has resolvers.
  5. A resolver receives the matched route, so it can read params such as route.paramMap.get('profileId').
  6. NavigationEnd means success; NavigationCancel often means a guard redirected or canceled.
  7. Always unsubscribe from router event subscriptions, usually with takeUntilDestroyed.
