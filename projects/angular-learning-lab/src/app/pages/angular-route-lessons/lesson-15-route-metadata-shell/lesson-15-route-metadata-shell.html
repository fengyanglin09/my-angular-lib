Lesson 11 vs Lesson 15

Lesson 11
A routed component reads its own route data.
Lesson 15
A parent shell reads the active child route metadata.
Why it matters
The shell can keep breadcrumbs and page chrome consistent.

Route Tree Detail

Many possible children
A parent route can define overview, billing, and settings.
One active branch
The current URL activates one primary child branch, so firstChild points to that active child.
Why deepest?
The deepest active route is usually the exact page, while parent data is broader section context.

Mental Model

URL changes
  router activates child route
  parent shell finds deepest child
  shell reads title + data
  breadcrumb and page chrome update

What To Remember

  1. Route metadata can drive shared layout, not just the routed component.
  2. The parent route can walk through firstChild to find the deepest active child route.
  3. NavigationEnd is a practical time to refresh breadcrumb and title UI.
  4. snapshot.data gives the active route's static metadata and resolved values.
  5. snapshot.title gives the active route title metadata when the route declares title.
  6. Use this for app chrome; use resolvers or services for backend records.
