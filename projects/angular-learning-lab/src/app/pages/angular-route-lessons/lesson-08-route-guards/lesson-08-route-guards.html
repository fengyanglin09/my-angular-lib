Route Guards

Route guards decide whether navigation should continue before Angular activates a route. A guard can allow the route, block it, or redirect to a safer route.

Guard Return Values

true
Allow navigation and activate the target route.
false
Cancel navigation and stay where you are.
UrlTree
Cancel the attempted route and redirect to the UrlTree route.

Mental Model

navigation starts
  router finds target route
  canActivate guard runs
  true -> activate component
  UrlTree -> redirect elsewhere

guard protects entry
  not every future state change

What To Remember

  1. canActivate runs before Angular activates the guarded route.
  2. A functional guard can use inject(...) to read services.
  3. Return true to allow navigation.
  4. Return false to cancel navigation and stay on the current route.
  5. Return a UrlTree when you want to redirect instead of merely canceling.
  6. A guard redirect can be absolute, or relative when you pass relativeTo: route.
  7. Relative guard redirects depend on the route snapshot passed into the guard.
  8. Guards run during navigation; they do not continuously watch access state after activation.
