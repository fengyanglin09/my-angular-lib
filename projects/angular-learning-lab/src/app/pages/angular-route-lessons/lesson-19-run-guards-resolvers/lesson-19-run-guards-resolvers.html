What To Compare

Default route
Changing :reportId reruns the resolver; changing only ?tab does not.
Query-aware route
paramsOrQueryParamsChange reruns the resolver for path or query changes.
Same component
The component can stay mounted while inputs and resolver data update.

Mental Model

URL changes
  router decides route is still same component
  route params/query params update
  runGuardsAndResolvers decides:
    rerun resolver?
    keep old resolver data?

What To Remember

  1. The default policy reruns guards and resolvers when route params change.
  2. Changing only query params may update component inputs without rerunning a resolver.
  3. Use runGuardsAndResolvers: 'paramsOrQueryParamsChange' when resolver data depends on query params.
  4. Do not rerun expensive resolvers on every query change unless the data truly depends on those query values.
  5. This setting belongs on the route record that owns the guard or resolver.
