Named Outlets

A component can have one primary unnamed router-outlet and additional named outlets. This is useful when the URL should control both the main content and a side panel, modal, drawer, or inspector.

Outlet Events

  1. (activate) runs after navigation makes Angular insert a routed component into that outlet.
  2. The $event value is the component instance that was activated.
  3. (deactivate) runs after navigation makes Angular remove the current routed component from that outlet.
  4. Opening sidePanel: ['notes'] triggers activate for the notes panel.
  5. Closing sidePanel: null triggers deactivate for the current side panel component.
  6. Primary content is normally changed by navigating to another primary route, not by closing it with null.
  7. This lesson uses those events only for display state and logging; the route itself is still the source of truth.

What To Remember

  1. Use one unnamed outlet for normal primary content.
  2. Use named outlets for secondary route-driven UI, like a side panel.
  3. The route config must include outlet: 'sidePanel' for the named outlet route.
  4. Closing a named outlet uses sidePanel: null in the outlets command.
  5. The primary outlet is main page content. Change it with another primary route, such as primary: ['dashboard'].
  6. activate and deactivate are outlet lifecycle events, not route definitions.
