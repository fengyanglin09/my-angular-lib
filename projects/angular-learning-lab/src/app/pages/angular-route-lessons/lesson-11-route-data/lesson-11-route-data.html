Route Data and Titles

Routes can carry static metadata with data and page titles with title. This is useful for page labels, breadcrumbs, access hints, and analytics names that belong to the route, not to a backend record.

Metadata Use Cases

Breadcrumbs
Route data can provide a stable label for navigation trails.
Access hints
Route data can describe what role a route expects.
Page titles
title updates the document title for the active route.

Data vs Resolver

data
Already known when writing the route config.
resolve
Loaded or calculated during navigation.
component reads
Both are available through ActivatedRoute.data.

What To Remember

  1. data is static metadata on the route config.
  2. title is the route's document title metadata.
  3. A component reads route metadata from ActivatedRoute.data.
  4. The same component can render different route metadata for different paths.
  5. Use data for route-owned facts, not backend records or form state.
  6. Resolved values also appear in ActivatedRoute.data, but they are produced during navigation.
