loadComponent vs loadChildren

loadComponent
Best for one standalone page component.
loadChildren
Best for a feature route file with several child routes.
Both
Use dynamic import(...), so Angular can split code.

What To Remember

  1. loadChildren lazy-loads a route group, not just one component.
  2. The lazy route file exports a normal Routes array.
  3. The parent route still needs a router-outlet for lazy children.
  4. Lazy loading is about when code is requested, not about changing how routes match.
  5. Use lazy route groups for larger feature areas that users may not visit right away.
