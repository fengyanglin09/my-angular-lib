CanDeactivate

canDeactivate guards protect leaving a route. They are useful for editors and forms where accidental navigation could lose unsaved changes.

Guard Return Values

true
Allow leaving the current route.
Promise<boolean>
Pause navigation until the user chooses what should happen.
false
Cancel navigation and keep the current route active.

When To Use canDeactivate

Main question
Is it safe to leave the current page?
Use for
Unsaved forms, draft editors, active uploads, unfinished wizard steps.
Not for
Login or role access before entering a page; use canActivate or canMatch.

canDeactivate Sequence

  1. The user is already on the editor route.
  2. Navigation tries to leave the editor route.
  3. canDeactivate asks the current component if leaving is safe.
  4. If allowed, Angular deactivates the editor and continues navigation.
  5. If denied, Angular keeps the current editor route active.

Mental Model

navigation tries to leave editor
  router calls canDeactivate guard
  guard asks component if it can leave
  true -> continue navigation
  Promise -> wait for the user's choice
  false -> stay on editor

What To Remember

  1. canDeactivate runs when navigation tries to leave the guarded route.
  2. The guard receives the current component instance.
  3. Return true to allow leaving.
  4. Return Promise<boolean> when the user needs to make a save/discard/stay decision.
  5. Resolve true to continue the original navigation, or false to stay.
  6. Use canDeactivate for leaving a route, not for deciding whether the target route should be entered.
  7. Use canActivate for entering a matched page that requires access.
  8. Use canMatch when the route branch itself should not match unless access is valid.
  9. Use this pattern carefully; the dialog should protect the user, not trap them.
