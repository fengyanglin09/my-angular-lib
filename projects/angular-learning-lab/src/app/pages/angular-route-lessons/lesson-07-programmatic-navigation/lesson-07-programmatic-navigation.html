Method Choice

navigate
Best when you want route commands, relative navigation, or query param options.
navigateByUrl
Best when you already have the complete URL string.
relativeTo
Keeps commands scoped under the current route, like a relative routerLink.

What To Remember

  1. Use Router.navigate when TypeScript needs to start navigation.
  2. relativeTo makes route commands resolve under the current activated route.
  3. The current path is the URL path before ?, such as /lesson-07-programmatic-navigation/archive.
  4. Navigation options are the object passed after the commands, such as queryParams and queryParamsHandling.
  5. router.navigate([]) keeps the current path and only applies navigation options.
  6. queryParamsHandling: 'merge' keeps existing query params unless you replace them.
  7. Use navigateByUrl when you already have a complete URL string.
