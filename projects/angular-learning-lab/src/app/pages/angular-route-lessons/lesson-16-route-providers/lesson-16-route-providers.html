Where Is The Service Created?

Route providers
Created for the matched route branch and available to its children.
Component providers
Created for one component subtree, even if the route is the same.
Root providers
Created once for the app and shared globally.

Mental Model

root provider
  whole app shares it

route provider
  parent route gets an injector
  child routes share that injector
  leaving the branch releases it

What To Remember

  1. Route records can declare providers.
  2. The route component and child routes can inject services from that route provider.
  3. Child routes under the same parent route share the same route-scoped service instance.
  4. A root service uses providedIn: 'root' and behaves like a global singleton.
  5. Route providers are useful for feature-local state that should reset when the feature is left.
  6. Do not make state global just because multiple child routes need it.
