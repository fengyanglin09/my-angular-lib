Preloading Lazy Routes

Lazy routes split code into separate chunks. Preloading lets Angular quietly download selected lazy chunks after the app is running, so a likely-next route can feel faster when the user opens it.

Decision Point

Preload this route
The strategy calls and returns load().
Skip this route
The strategy returns of(null) instead.
loadChildren
Creates a lazy route chunk.
withPreloading(...)
Tells the router how to preload lazy chunks after navigation.
data.preload
Lets each route opt in without hard-coding route names in the strategy.

Why Not Just Make It Eager?

Eager route
Included in the initial app load, even if the user never opens it.
Lazy + preload
Kept out of the first bundle, then downloaded after the app is already running.
Selective preload
Can be based on route data, role, network, or likely next navigation.

Mental Model

lazy route
  code is split

preloading strategy
  sees lazy route
  checks route data
  calls load() only when allowed

user clicks later
  route can activate faster

What To Remember

  1. loadChildren makes a route lazy.
  2. withPreloading controls whether lazy routes are downloaded early.
  3. A custom PreloadingStrategy receives each lazy route and a load() function.
  4. Calling load() preloads the lazy route chunk.
  5. Returning of(null) skips preloading for that route.
  6. Route data is a clean way to mark preload candidates.
