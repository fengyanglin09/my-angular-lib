Redirects And Wildcards

Redirects move one matched URL to another route. Wildcards catch unmatched paths. Together they help you create default child pages, support old URLs, and show useful fallback content when a path is missing.

What To Remember

  1. redirectTo changes navigation to another route.
  2. pathMatch: 'full' is important for empty-path redirects.
  3. ** should usually be the last route in its route group.
  4. A wildcard can be scoped inside child routes, so it only catches missing paths in that section.
