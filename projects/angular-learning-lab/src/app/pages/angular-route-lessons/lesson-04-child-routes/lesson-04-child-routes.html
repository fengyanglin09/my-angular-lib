Mental Model

parent route matches
  parent component renders
  parent template has router-outlet
  child route matches
  child component renders inside parent

routerLink="activity"
  no leading slash
  relative to the parent route

routerLink="/activity"
  leading slash
  starts from app root

What To Remember

  1. children defines routes below a parent route.
  2. The parent component stays visible while child route content changes.
  3. A nested router-outlet is where child components render.
  4. Links like routerLink="activity" are relative to the current parent route.
  5. A leading slash changes the meaning: routerLink="/activity" starts from the app root.
  6. Angular knows which outlet to use from the matched route tree: child routes render in their parent's outlet.
