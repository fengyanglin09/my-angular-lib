Name Matching And Precedence

Matching rule
The component input name must match the route value key exactly.
Examples
:workspaceId binds to workspaceId; data.accessLevel binds to accessLevel.
Sources
Query params, path params, static data, and resolver data can all bind.
Highest precedence
Resolver data wins if multiple sources use the same key.

Mental Model

router activates component
  reads params
  reads query params
  reads route data
  runs resolvers
  sets matching component inputs

What To Remember

  1. withComponentInputBinding() is enabled in provideRouter(...).
  2. The component input name must match the route param, query param, data key, or resolver key.
  3. Route params bind to inputs with matching names.
  4. Query params bind to inputs too, but missing query params become undefined.
  5. Static route data and resolver data can bind to component inputs.
  6. Resolver data has the highest precedence when keys collide.
  7. Use this when you only need route values; use ActivatedRoute when you need route tree APIs or custom stream logic.
