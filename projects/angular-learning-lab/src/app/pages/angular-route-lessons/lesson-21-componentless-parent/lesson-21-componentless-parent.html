Why Not Add A Parent Component?

Use componentless
When the parent only groups config and has no UI to render.
Use a parent component
When the parent needs layout, tabs, sidebars, loading state, or a visible shell.
Practical result
You avoid throwaway components whose only job is route grouping.

What To Remember

  1. A route can have children without having a component.
  2. That route is useful for grouping shared params, data, guards, or providers.
  3. The componentless route does not render its own UI.
  4. Child routes below it still render their components in the nearest parent router-outlet.
  5. Use a real parent component when the parent route needs visible layout.
