Route Resolvers

A resolver loads data during navigation before Angular activates the route. The component receives that data from ActivatedRoute.data, so the page can start with required data already available.

Mental Model

navigation starts
  router matches route
  resolver runs
  resolved data is attached to route
  component activates
  component reads ActivatedRoute.data

What To Remember

  1. ResolveFn runs before route activation.
  2. A resolver receives the matched route, so it can read route params like route.paramMap.get('projectId').
  3. The route config chooses the data key, such as project in the resolve object.
  4. A route can define multiple resolvers; each key becomes one value on ActivatedRoute.data.
  5. The component reads resolved values from ActivatedRoute.data.
  6. Resolvers are good for required route data, but they delay route activation while loading.
  7. Optional or secondary data can usually load after activation from the component or a service.
