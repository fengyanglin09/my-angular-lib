What To Remember

  1. GET reads data; POST usually creates data.
  2. The create request body usually does not include the server-generated id.
  3. The response body should include the created object, including fields generated by the backend.
  4. A fake API can teach the same component flow without writing files or running a backend.
  5. In a real app, the API service method would use http.post<CourseDto>(url, draft).
