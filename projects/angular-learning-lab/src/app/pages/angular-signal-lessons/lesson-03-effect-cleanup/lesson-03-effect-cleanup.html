Effect Cleanup

Effects can start work that keeps running after the effect function returns. Use onCleanup to cancel the previous work before the effect reruns or before the component is destroyed.

What To Remember

  1. effect((onCleanup) => ...) gives you a cleanup hook.
  2. Cleanup runs before the effect reruns for changed dependencies.
  3. Cleanup also runs when Angular destroys the effect with the component.
  4. Use cleanup for timers, subscriptions, browser listeners, and pending async work.
