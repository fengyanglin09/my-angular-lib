Immutable Updates

Signals notify dependents when you call set or update with a meaningful new value. For object and array state, that usually means creating new references instead of mutating the existing object.

How To Try It

  1. Click the safe buttons. The computed cart view, total, and effect log update together.
  2. Click unsafe direct mutation. The lesson logs the click, but the cart computed values do not rerun from that mutation.
  3. Click set same cart reference. Calling set with the same object reference still does not notify dependents.
  4. Use immutable updates for signal state you expect Angular to react to.
