untracked And Equality

untracked lets an effect read a signal without depending on it, while a signal equality function controls whether a new value should notify dependents. Use both sparingly, when default reactivity is too noisy.

What To Remember

  1. Effects track the signals they read.
  2. untracked(...) is for incidental reads that should not become dependencies.
  3. Custom equality is useful when new object references may represent the same meaningful value.
  4. Prefer default signal behavior unless you have a specific noisy dependency problem.
