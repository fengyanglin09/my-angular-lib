Input Signals

Signal inputs let a child component receive parent data as signals. The parent still owns the state, and the child can read the input with plan() or use it inside computed.

Mental Model

parent
  owns writable signals
  passes values to child

child
  declares input signals
  reads plan()
  computes derived values

data flow
  parent -> child

What To Remember

  1. input(...) creates an optional input signal with a default value.
  2. input.required<T>() creates a required input signal.
  3. Read input signals by calling them, like plan().
  4. The child should not mutate parent state directly; data still flows parent to child.
