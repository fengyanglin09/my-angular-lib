Model Signals

model(...) creates a signal input that the child can update. It is useful when a parent owns state, but a child control needs to change that state through two-way binding.

Mental Model

input()
  parent -> child
  child reads value

model()
  parent <-> child
  child can update value

parent signal remains source of truth

What To Remember

  1. model(...) creates a writable signal input.
  2. The parent can bind to it with [(quantity)]="quantity".
  3. The child can read and update the model signal.
  4. Use input for one-way data, and model when the child should update the parent-owned value.
