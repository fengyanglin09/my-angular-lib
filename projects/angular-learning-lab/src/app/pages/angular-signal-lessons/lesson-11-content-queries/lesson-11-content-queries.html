Content Queries

contentChild and contentChildren let a reusable child component inspect content projected into it with ng-content. This is useful when building cards, tabs, menus, layout shells, and form components that accept custom content from a parent.

Mental Model

parent template
  provides projected content

child component
  renders ng-content
  reads contentChild()
  reads contentChildren()

content query
  sees projected content
  not the child's own view

What To Remember

  1. viewChild reads the component's own template.
  2. contentChild reads content projected into the component.
  3. Querying a directive returns the directive; adding read: ElementRef returns the host element.
  4. contentChildren updates when projected matches are added or removed.
  5. The parent owns projected content; the child component can observe it to build flexible APIs.
