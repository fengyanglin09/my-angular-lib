When To Reach For Ramda

Prefer the simplest readable option

Optional chaining is usually clearer for one fixed path. Ramda starts to help when the same nested path is reused, passed into helpers, or combined with other transformations.

What To Remember

  1. path reads a nested value and returns undefined when missing.
  2. pathOr reads a nested value with a fallback.
  3. lensPath creates a reusable focus on one nested path.
  4. view reads through a lens.
  5. set replaces the focused value in a copied object.
  6. over updates the focused value with a function in a copied object.
