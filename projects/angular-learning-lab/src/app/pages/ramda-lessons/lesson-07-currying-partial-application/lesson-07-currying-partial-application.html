Currying And Partial Application

Ramda functions are designed to be configured in steps. Currying lets you provide some arguments now and get a new reusable function back for the remaining arguments. Ramda fills those arguments from left to right, so put reusable configuration first and the changing data last.

What To Remember

  1. Currying turns one multi-argument function into a function that can receive arguments in steps.
  2. Ramda curry fills arguments from left to right, so argument order is part of the design.
  3. Partial application means pre-filling some leading arguments to create a smaller function.
  4. partial can wrap a normal function; the original function does not need to be curried first.
  5. This is useful when the early arguments are configuration and the last argument is the data.
  6. Plain TypeScript wrapper functions are often clearer for small one-off cases.
  7. Ramda becomes more useful when the configured function is reused in maps, filters, pipes, or selectors.
