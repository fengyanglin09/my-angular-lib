# Angular Layout / UI Architecture Lesson Summary

Angular Layout lessons focus on application structure: shell layouts, navigation
regions, feature folders, reusable page layouts, design consistency, and common
dashboard/detail workflows.

| Lesson | Route | Teaches |
| --- | --- | --- |
| 1. Shell Layout | `/angular-layout-lessons/lesson-01-shell-layout` | How a stable app shell wraps routed feature pages. |
| 2. Sidenav And Topnav | `/angular-layout-lessons/lesson-02-sidenav-topnav` | How to separate global navigation, feature navigation, and page actions. |
| 3. Feature Folders | `/angular-layout-lessons/lesson-03-feature-folders` | How to group routes, pages, components, services, and models by feature. |
| 4. Reusable Page Layouts | `/angular-layout-lessons/lesson-04-reusable-page-layouts` | How page layout components use slots for repeated structure. |
| 5. Design Consistency | `/angular-layout-lessons/lesson-05-design-consistency` | How shared primitives and page patterns prevent CSS drift. |
| 6. Dashboards And Detail Pages | `/angular-layout-lessons/lesson-06-dashboards-detail-pages` | How dashboards summarize and detail pages support focused inspection/action. |

## Review Order

- Start with lessons 1-2 for shell/navigation structure.
- Use lessons 3-5 for maintainable app organization and consistent page design.
- Use lesson 6 when designing dashboard-to-detail workflows.
