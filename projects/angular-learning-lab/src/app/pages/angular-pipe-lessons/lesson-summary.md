# Angular Pipe Lessons

This track teaches pipes as template display transformers. Pipes should make rendered values easier to read without taking ownership of application state.

| Lesson                                | Route                                                            | What it teaches                                                                  |
| ------------------------------------- | ---------------------------------------------------------------- | -------------------------------------------------------------------------------- |
| 1. Built-in Pipes                     | `/angular-pipe-lessons/lesson-01-built-in-pipes`                 | Use Angular's built-in pipes for common display formatting.                      |
| 2. Custom Pure Pipes                  | `/angular-pipe-lessons/lesson-02-custom-pure-pipes`              | Create a focused, reusable pure pipe.                                            |
| 3. Pipe Parameters                    | `/angular-pipe-lessons/lesson-03-pipe-parameters`                | Pass parameters into a pipe to configure formatting.                             |
| 4. Pure Vs Impure Pipes               | `/angular-pipe-lessons/lesson-04-pure-vs-impure-pipes`           | Understand when pure pipes run and why impure pipes are expensive.               |
| 5. Async Pipe                         | `/angular-pipe-lessons/lesson-05-async-pipe`                     | Subscribe to Observables in templates and clean up automatically.                |
| 6. Formatting API Data                | `/angular-pipe-lessons/lesson-06-formatting-api-data`            | Format backend DTO values for display without changing source data.              |
| 7. Performance Considerations         | `/angular-pipe-lessons/lesson-07-performance-considerations`     | Avoid expensive pipe work and keep transformations cache-friendly.               |
| 8. Pipe Vs Computed / Signal / Method | `/angular-pipe-lessons/lesson-08-pipe-vs-computed-signal-method` | Choose the right transformation location for display values and stateful values. |
