Rate Limiting With auditTime

The auditTime operator is useful for noisy UI events. It watches a time window and emits the latest value at the end of that window.

What To Remember

  1. auditTime emits the latest value after each time window.
  2. debounceTime waits for quiet before emitting.
  3. throttleTime emits early, then ignores values for a while.
  4. Use auditTime for noisy events where the latest value matters regularly, such as resize, scroll, or drag updates.
