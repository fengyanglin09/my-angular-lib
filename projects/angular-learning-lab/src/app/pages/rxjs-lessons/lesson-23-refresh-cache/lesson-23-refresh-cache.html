What To Remember

  1. refresh$ is only a trigger. It does not store the profile.
  2. startWith creates the initial load for the first subscriber.
  3. switchMap turns each refresh trigger into a new backend request.
  4. shareReplay keeps the latest loaded state for late subscribers.
