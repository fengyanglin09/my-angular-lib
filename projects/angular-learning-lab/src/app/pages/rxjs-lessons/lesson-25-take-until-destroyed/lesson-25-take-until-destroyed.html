takeUntilDestroyed In Practice

Some streams are subscribed to in TypeScript because they update local signals, call imperative APIs, or connect to route params. Use takeUntilDestroyed so Angular owns the cleanup when the component is destroyed.

Mental Model

component created
  subscriptions start

route param changes
  same subscription receives new value
  component is not destroyed

leave this lesson
  component is destroyed
  takeUntilDestroyed unsubscribes

What To Remember

  1. takeUntilDestroyed is for subscriptions you create in TypeScript.
  2. Route param changes usually reuse the same component instance.
  3. Intervals and component-owned Subjects can keep running if not cleaned up.
  4. The async pipe already cleans up template subscriptions for you.
