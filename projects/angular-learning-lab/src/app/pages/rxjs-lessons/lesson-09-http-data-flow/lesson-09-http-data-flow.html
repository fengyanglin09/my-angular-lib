What To Remember

  1. A reload stream can represent events that should fetch fresh backend data.
  2. switchMap is common for reloads because newer requests replace older ones.
  3. catchError keeps the outer view-model stream alive after a failed request.
  4. shareReplay is useful when multiple subscribers should share the latest loaded value.
