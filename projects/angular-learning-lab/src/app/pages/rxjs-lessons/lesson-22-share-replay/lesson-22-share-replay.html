Sharing Work With shareReplay

Cold Observables run once per subscriber. The share and shareReplay operators let subscribers reuse work, but only shareReplay replays cached values to late subscribers.

Mental Model

Subscriber B joins while request #1 is running.
Subscriber C joins after request #1 completed.

cold source
  subscriber A -> request #1
  subscriber B -> request #2
  subscriber C -> request #3

share
  subscriber A/B share request #1
  subscriber C starts request #2

shareReplay
  subscriber A/B share request #1
  subscriber C receives cached request #1 value

What To Remember

  1. share shares one active subscription to the source.
  2. shareReplay shares the source and caches recent emissions.
  3. bufferSize: 1 means replay the latest value.
  4. refCount: true disconnects from the source when no subscribers remain.
