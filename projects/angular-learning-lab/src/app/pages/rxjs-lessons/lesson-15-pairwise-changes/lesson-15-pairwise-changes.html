Comparing Changes With pairwise

The pairwise operator waits until it has two values, then emits them as [previous, current]. It is useful when the app needs to react to transitions, not just individual values.

What To Remember

  1. pairwise does not emit for the first source value.
  2. It emits after the second value because it finally has a pair.
  3. startWith can provide an initial previous value.
  4. Use it when the transition matters: previous route to current route, old status to new status, or previous score to current score.
