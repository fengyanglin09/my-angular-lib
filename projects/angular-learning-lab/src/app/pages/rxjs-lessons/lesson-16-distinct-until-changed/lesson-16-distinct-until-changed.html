Avoiding Duplicate Work With distinctUntilChanged

The distinctUntilChanged operator blocks consecutive duplicate values. In Angular apps, it helps avoid duplicate backend requests when route params, filters, or search values did not meaningfully change.

    @for (log of rawLogs(); track $index) {

What To Remember

  1. By default, distinctUntilChanged uses ===.
  2. For objects, pass a comparer function that checks meaningful fields.
  3. It only compares the current value to the previous emitted value.
  4. Use map before it when values should be normalized first.
