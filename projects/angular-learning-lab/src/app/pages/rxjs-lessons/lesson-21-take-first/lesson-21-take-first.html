One-Time Reads With take and first

The take(1) and first() operators are useful when a stream should give you one value and then stop. This is common for dialog results, one-time route reads, and short-lived workflow decisions.

Difference

take(1)
  - takes the next value
  - completes if source completes empty

first()
  - takes the first matching value
  - completes after that match
  - errors if no value arrives

What To Remember

  1. take(1) is the simple choice for “give me the next value once.”
  2. first(predicate) emits the first matching value, then completes.
  3. take(1) completes quietly if the source completes without values.
  4. first() errors if the source completes before a matching value appears.
