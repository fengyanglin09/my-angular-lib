Prevent Duplicate Submits

Mental Model

All buttons emit into the same submitClicks$ stream.
All submit clicks go through exhaustMap.

click #1
  exhaustMap accepts it
  backend request starts

click #2 while request #1 is active
  exhaustMap ignores it

request #1 completes
  next click can be accepted

What To Remember

  1. exhaustMap subscribes to the first inner Observable.
  2. While that inner Observable is active, new source emissions are ignored.
  3. After the inner Observable completes, the next source emission can start new work.
  4. This is different from switchMap, which cancels the previous request.
