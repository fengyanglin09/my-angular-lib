Accumulating State With scan

The scan operator turns a stream of events into a stream of state. It is useful when the next value depends on previous values, such as carts, notification logs, undo history, selected rows, or paginated results.

What To Remember

  1. scan emits after every source event.
  2. Each emission becomes the previous state for the next event.
  3. It is similar to a reducer, but it keeps emitting over time.
  4. Use it when an RxJS stream needs memory without introducing global state.
