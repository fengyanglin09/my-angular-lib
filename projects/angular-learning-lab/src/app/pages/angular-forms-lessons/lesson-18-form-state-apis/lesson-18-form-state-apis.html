What To Remember

  1. touched is useful for validation messages after blur or submit.
  2. dirty means the value has changed from a clean state.
  3. pristine is the opposite of dirty.
  4. pending is useful while async work is unresolved.
  5. disable removes a control from form.value; getRawValue still includes it.
  6. reset restores values and clears dirty/touched state.
