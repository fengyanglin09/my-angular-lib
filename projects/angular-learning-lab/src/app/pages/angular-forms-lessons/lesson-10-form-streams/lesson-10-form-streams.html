What To Remember

  1. valueChanges emits future value changes; it does not emit the current value on subscribe.
  2. Use getRawValue() when you need the current form value right now.
  3. statusChanges emits future status changes; it does not emit the current status on subscribe.
  4. Use form.status when you need the current status right now.
  5. Use startWith when a combined stream needs the current form state immediately.
  6. Use debounceTime before autosave, search, or preview work.
  7. Use takeUntilDestroyed for component-owned subscriptions.
