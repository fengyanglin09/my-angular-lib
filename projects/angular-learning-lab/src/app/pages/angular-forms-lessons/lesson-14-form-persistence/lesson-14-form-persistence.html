Restore brings back the typed values. It does not bring back the old screen moment, such as which fields were touched or whether the user had already tried to submit.

Data Vs Screen State

Draft data comes back; interaction state starts fresh.

Values like name, email, role, and notes are worth saving. Flags like dirty, touched, focused field, validation messages, and submit attempts usually belong to the current visit.

What To Remember

  1. Persist typed values, not the old screen interaction state.
  2. Use getRawValue() when saving a complete draft payload.
  3. Debounce autosave so typing does not write storage on every keystroke.
  4. Use reset(savedValue, {{ '{' }} emitEvent: false {{ '}' }}) so restored values feel like a freshly loaded form without triggering autosave.
  5. Use emitEvent: false when reset or restore should not trigger autosave.
  6. Keep storage code in a service so the component stays focused on form behavior.
