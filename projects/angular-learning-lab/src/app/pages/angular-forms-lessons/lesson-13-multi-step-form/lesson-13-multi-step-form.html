Multi-step Forms

A wizard does not need separate forms for each screen. One typed FormGroup can own all values, while the current step decides which nested group is visible and validated before moving forward.

What To Remember

  1. Use one parent FormGroup for the whole wizard when all steps belong to one payload.
  2. Use nested groups to model each step.
  3. Validate the current step group before moving forward.
  4. Going backward usually should not require validation.
  5. Submit with getRawValue() so the final payload is complete.
