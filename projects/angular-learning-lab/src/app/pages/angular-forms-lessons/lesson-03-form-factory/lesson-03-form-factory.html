What To Remember

  1. Use plain objects for data that comes from an API, route resolver, local storage, or defaults.
  2. Use a field object when the same metadata should drive controls and template rendering.
  3. Use a form factory to loop over that field object and create controls.
  4. patchValue(...) updates existing controls without rebuilding the form.
  5. reset(...) restores values and also resets form state such as dirty and touched.
